## 4. Conduct Experiment

-   Execute experiments in order

**Robustness** - Return any errors upon failure or time-out - Fix the
code - Re-attempt up to four times

**Iteration** - After completion of each experiment: - Receive results -
Take notes in the style of an experimental journal - Conditional on the
results: - Re-plan and implement the next experiment - Repeat process up
to five times - Full history of execution must remain visible

**Optimization & Evolution** - Repeating own error to improve output -
Refine through trial-error & simulation - Generate reward (self
rewarding) - Debate, critique, and defend position - Competition to
uncover & patch vulnerability - Validate externally to refine ground (?)
